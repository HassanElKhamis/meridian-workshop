1. System Overview

The Meridian Inventory Dashboard is a single-page application for monitoring component inventory, orders, demand forecasts, and spending across multiple warehouses. The application runs locally without a database — all data is stored in JSON files and loaded into memory at server startup.

3. Backend Architecture

3.1 FastAPI Application

Single-file application (server/main.py, ~310 lines) with Pydantic response models. Data loaded at startup via server/mock_data.py into in-memory lists.

Helper Functions

  • apply_filters(items) — Filters lists by warehouse, category, and status query params
  • filter_by_month(data, month) — Time-based filtering with quarter support ("q1"–"q4")

5. Data Layer

All data is stored as static JSON files in server/data/. Files are loaded once at module import time via mock_data.py. There is no database, no persistence layer, and no write-back mechanism (POST endpoints exist for tasks and POs but are in-memory only).

6. Findings and Known Limitations

🔴 Reports API Client Gap Critical

Reports.vue imports axios directly and hardcodes http://localhost:8001/api/reports/... URLs. Other views use the shared api.js client. The Reports endpoints are missing from api.js entirely. This breaks the abstraction and would make environment configuration (staging/production) unreliable for the Reports page.

🔴 Restocking Feature Missing R2

No /restocking route exists. No backend endpoint for restocking recommendations. The RFP requires a new view that recommends purchase orders given stock levels, demand forecasts, and a budget ceiling. This is a greenfield build.

🟡 Console Log Noise R1

Reports.vue has ~15 console.log statements scattered through loadData(), formatNumber(), formatMonth(), and getBarHeight(). These are debug artifacts left by the previous vendor and pollute the browser console in production.

🟡 Incomplete i18n D3

Reports page labels ("Quarterly Performance", "Monthly Revenue Trend", "Month-over-Month Analysis") are hardcoded English strings with no translation keys. The "Reports" nav tab in App.vue also uses a hardcoded string. Japanese locale file has gaps.

🟡 Mixed API Styles Technical Debt

App.vue uses Options API (data(), methods, mounted) while Dashboard.vue and other views use Composition API with setup(). The handoff doc notes a Composition API migration was started but never completed. This creates an inconsistent codebase and hinders refactoring.

🟡 Orphaned Component Observation

Backlog.vue exists in src/views/ but is NOT registered in the router. It is unreachable from the application navigation. The Dashboard page renders backlog data inline, so Backlog.vue may be a remnant of an abandoned separate page.

🟡 Tasks API Mismatch Observation

The frontend calls /api/tasks endpoints (GET, POST, DELETE, PATCH) but these are not documented in the vendor handoff. The backend serves them but they operate on in-memory data only — tasks are lost on server restart.

🟢 No Browser Tests R3

The application has zero automated browser tests. The RFP requires Playwright-based test coverage for critical flows. A Playwright MCP server is configured in the repo and ready to use. This is a greenfield effort.
